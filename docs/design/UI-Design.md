# Trusted Group UI/UX Design

## 1. Design Philosophy

### 1.1 Core Principles
- **Community First:** The interface emphasizes the specific "Trusted Group" (e.g., Church) context over the general platform.
- **Trust & Accountability:** Ledger balances and transaction histories are prominent to encourage fairness.
- **Simplicity:** No maps, no complex reviews. Just lists, requests, and simple status updates.
- **Multi-Context:** Clear distinction between different communities (Organizations) a user belongs to.

### 1.2 Visual Identity (Trusted Theme)
*Inherits core system but with warmer/softer tones for "Group" feel.*

- **Primary:** `#2D7A3E` (Forest Green) - Continuity with brand.
- **Secondary:** `#F59E0B` (Amber) - Warmth.
- **Backgrounds:** Cleaner whites/grays to emphasize the list content.

## 8. Responsive Strategy

- **Adaptive Navigation:** Bottom navigation for mobile, side navigation for tablet+.

## 9. Data Format Guidelines

### 9.1 Date Display
**Backend Format:** YYYY-MM-DD (e.g., "2026-01-13")  
**Display Format:** User-friendly based on context
- **Short:** "Jan 13" (same year)
- **Medium:** "Jan 13, 2026" (different year or formal)
- **Relative:** "Today", "Yesterday", "2 days ago" (recent)

**Implementation:**
```kotlin
// Parse from backend
val date = LocalDate.parse("2026-01-13")

// Display formats
val short = date.format(DateTimeFormatter.ofPattern("MMM d"))
val medium = date.format(DateTimeFormatter.ofPattern("MMM d, yyyy"))
val relative = formatRelative(date) // "2 days ago"
```

### 9.2 Currency Display
**Backend Format:** int32 cents (e.g., 550 = $5.50)  
**Display Format:** "$X" or "$X.XX"
- **Whole dollars:** "$5"
- **With cents:** "$5.50"

**Implementation:**
```kotlin
fun formatCurrency(cents: Int): String {
    val dollars = cents / 100.0
    return if (cents % 100 == 0) {
        "$${cents / 100}"
    } else {
        "$${"%.2f".format(dollars)}"
    }
}
```
